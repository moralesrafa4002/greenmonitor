# Fase 1 — Planeación ágil con Scrum

## Objetivo de la fase

Organizar el desarrollo del sistema mediante una metodología ágil que permita dividir el trabajo en entregables incrementales. Esta fase define el Product Backlog, las historias de usuario, las tareas por sprint y las evidencias esperadas.

## Qué se hizo

Se definió una planeación Scrum adaptada al proyecto académico. El sistema se dividió en cuatro sprints principales:

1. **Sprint 1:** análisis del problema y diagramas de contexto/DFD.
2. **Sprint 2:** modelo de datos, diccionario de datos y modelo ER.
3. **Sprint 3:** creación de base de datos y reglas de negocio.
4. **Sprint 4:** interfaz web local, pruebas y documentación final.

## Por qué se hizo así

El proyecto tiene varias partes obligatorias: análisis estructurado, modelos, base de datos, interfaz web y documentación. Scrum permite trabajar por incrementos, revisar avances y corregir el proyecto antes de llegar al entregable final.

## Roles sugeridos

| Rol Scrum | Responsabilidad en el proyecto |
|---|---|
| Product Owner | Define necesidades del invernadero y prioriza funcionalidades. |
| Scrum Master | Organiza el trabajo, elimina bloqueos y mantiene el avance. |
| Equipo de desarrollo | Diseña modelos, base de datos, código, pruebas y documentación. |
| Usuario final | Consulta registros, visualiza alertas y analiza condiciones. |

## Definición de terminado

Una fase se considera terminada cuando:

- Tiene documentación clara.
- Tiene archivos fuente editables.
- Tiene evidencia o resultado verificable.
- Se conecta de forma coherente con las demás fases.
- Puede ser explicada por el equipo durante una exposición.
