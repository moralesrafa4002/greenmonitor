# Planeación por sprints

## Sprint 1 — Análisis del problema

### Objetivo
Comprender el sistema, identificar actores, procesos principales y flujos de información.

### Tareas
- Definir objetivo general y alcance.
- Identificar actores externos.
- Elaborar diagrama de contexto.
- Elaborar DFD Nivel 0.
- Elaborar DFD Nivel 1.

### Entregable incremental
Documentación de análisis estructurado y diagramas iniciales.

## Sprint 2 — Modelado de datos

### Objetivo
Diseñar la estructura de información que permitirá almacenar registros ambientales, fuentes, reglas y alertas.

### Tareas
- Identificar entidades.
- Definir atributos.
- Definir llaves primarias y foráneas.
- Elaborar diccionario de datos.
- Elaborar modelo ER.
- Revisar normalización.

### Entregable incremental
Modelo ER, diccionario de datos y explicación de tablas.

## Sprint 3 — Base de datos y reglas

### Objetivo
Construir la base de datos y programar la lógica de reglas de negocio.

### Tareas
- Crear script MySQL.
- Crear script SQLite para ejecución local.
- Insertar datos de prueba.
- Crear vistas y consultas.
- Definir reglas de alerta.
- Validar integridad referencial.

### Entregable incremental
Base de datos funcional con datos de prueba y reglas documentadas.

## Sprint 4 — Interfaz web local y cierre

### Objetivo
Implementar una interfaz local para consulta, actualización y visualización de alertas.

### Tareas
- Crear servidor web local.
- Crear pantalla de dashboard.
- Crear pantalla de registros.
- Crear pantalla de alertas.
- Crear pantalla de estadísticas.
- Probar filtros.
- Documentar ejecución y despliegue por SSH.

### Entregable incremental
Sistema ejecutable localmente y documentación final completa.
