# Product Backlog

| ID | Historia de usuario | Prioridad | Criterio de aceptación |
|---|---|---:|---|
| HU-01 | Como administrador quiero registrar zonas del invernadero para identificar dónde se toman las mediciones. | Alta | El sistema permite distinguir zonas interiores y exteriores. |
| HU-02 | Como usuario quiero cargar registros ambientales para guardar temperatura, humedad, radiación, UV y ventilación. | Alta | El registro se almacena correctamente con fecha, hora y zona. |
| HU-03 | Como usuario quiero consultar registros históricos para analizar el comportamiento del invernadero. | Alta | Se muestran registros filtrables por fecha, zona y tipo. |
| HU-04 | Como administrador quiero definir reglas de negocio para generar alertas automáticas. | Alta | Las reglas se documentan y se aplican al registrar datos. |
| HU-05 | Como usuario quiero ver alertas activas para actuar antes de que el cultivo se afecte. | Alta | Las alertas activas se muestran en la interfaz web. |
| HU-06 | Como usuario quiero consultar estadísticas por hora y por día para analizar tendencias. | Media | El sistema calcula promedios, mínimos y máximos. |
| HU-07 | Como administrador quiero consultar la fuente de los datos externos para documentar su origen. | Media | La base registra la fuente de datos como sensor local, NASA u otra fuente. |
| HU-08 | Como profesor quiero revisar modelos de análisis para validar el diseño del sistema. | Alta | El repositorio incluye contexto, DFD0, DFD1, ER, UML y diccionario. |
| HU-09 | Como administrador técnico quiero desplegar el sistema localmente y acceder por SSH. | Media | Existe una guía de instalación local y administración por SSH. |
| HU-10 | Como equipo quiero tener documentación por fase para explicar qué se hizo y por qué. | Alta | Cada fase tiene README y archivos relacionados. |

## Priorización

Las historias de mayor prioridad son las que permiten cumplir los requisitos obligatorios del proyecto: modelado, base de datos, registros, alertas e interfaz. Las historias de prioridad media complementan el sistema con estadísticas, documentación de fuentes externas y despliegue.
